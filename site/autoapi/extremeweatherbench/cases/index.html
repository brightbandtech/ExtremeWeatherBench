    CaseOperator dataclass 

    CaseOperator(case_metadata, metric_list, target, forecast)

    A class which stores the graph to process an individual case.

    This class is used to store the graph to process an individual case. The purpose of this class is to be a one-stop-shop for the evaluation of a single case. Multiple CaseOperators can be run in parallel to evaluate multiple cases, or run through the ExtremeWeatherBench.run() method to evaluate all cases in an evaluation in serial.

    Attributes:

    • case_metadata (IndividualCase) –

      IndividualCase metadata

    • metric_list (Sequence[BaseMetric]) –

      A list of metrics that are to be evaluated for the case operator

    • target_config (Sequence[BaseMetric]) –

      A TargetConfig object

    • forecast_config (Sequence[BaseMetric]) –

      A ForecastConfig object

    IndividualCase dataclass 

    IndividualCase(
    case_id_number,
    title,
    start_date,
    end_date,
    location,
    event_type,
)

    Container for metadata defining a single or individual case.

    An IndividualCase defines the relevant metadata for a single case study for a given extreme weather event; it is designed to be easily instantiable through a simple YAML-based configuration file.

    Attributes:

    • case_id_number (int) –

      A unique numerical identifier for the event.

    • start_date (datetime) –

      The start date of the case, for use in subsetting data for analysis.

    • end_date (datetime) –

      The end date of the case, for use in subsetting data for analysis.

    • location (Region) –

      A Location dataclass representing the location of a case.

    • event_type (str) –

      A string representing the type of extreme weather event.

    load_individual_cases_from_yaml 

    load_individual_cases_from_yaml(yaml_file)

    Load IndividualCase metadata directly from a yaml file.

    This function is a wrapper around load_individual_cases that reads the yaml file directly. It is useful for loading cases from a yaml file that is not part of the ExtremeWeatherBench data package. Note that the yaml file must be in the same format as described in the ExtremeWeatherBench documentation; errors will be raised within the dacite.from_dict call if the yaml file otherwise.

    Example of a yaml file:

    cases:
  - case_id_number: 1
    title: Event 1
    start_date: 2021-01-01 00:00:00
    end_date: 2021-01-03 00:00:00
    location:
        type: bounded_region
        parameters:
            latitude_min: 10.0
            latitude_max: 55.6
            longitude_min: 265.0
            longitude_max: 283.3
    event_type: tropical_cyclone

    Parameters:

    • yaml_file (Union[str, Path]) –

      A path to a yaml file containing the case metadata.

    Returns:

    • IndividualCaseCollection –

      A collection of IndividualCase objects.
